## 🎯 当前 Sprint (v4.1.0)

### Phase 2 第一批：CBA + 电竞

#### P2-1: CBA 赛程集成 (3h)

**目标**: 支持中国男子篮球职业联赛赛程查询

**技术方案**:
1. 使用 Dashscope MCP 搜索 CBA 赛程
2. 解析比赛时间、对阵球队
3. 支持关注球队配置
4. 输出格式与 NBA 保持一致

**配置示例**:
```json
{
  "basketball": {
    "leagues": {
      "cba": {
        "name": "CBA",
        "enabled": true,
        "teams": ["广东宏远", "辽宁本钢", "北京首钢"]
      }
    }
  }
}
```

**输出格式**:
```
🏀 CBA
💚 ⭐⭐⭐⭐ 03-13 19:35 | 辽宁本钢 @ 广东宏远 | 咪咕体育
```

---

#### P2-2: 电竞赛程集成 (3h)

**目标**: 支持 LPL、LCK 春季赛赛程查询

**技术方案**:
1. 使用 Dashscope MCP 搜索电竞赛程
2. 解析比赛时间、对阵战队、BO 系列
3. 支持关注战队配置
4. 固定输出格式

**配置示例**:
```json
{
  "esports": {
    "name": "电竞",
    "enabled": true,
    "leagues": {
      "lpl": {
        "name": "LPL 春季赛",
        "enabled": true,
        "teams": ["WBG", "EDG", "BLG", "JDG"]
      },
      "lck": {
        "name": "LCK 春季赛",
        "enabled": true,
        "teams": ["GEN", "T1"]
      }
    }
  }
}
```

**输出格式**:
```
🎮 电竞
💚 ⭐⭐⭐⭐ 03-13 17:00 | WBG vs EDG | LPL 春季赛 | B 站
⭐⭐ 03-13 19:00 | BLG vs JDG | LPL 春季赛 | B 站
```

---

#### P2-3: 缓存系统实现 (2h)

**目标**: 减少重复搜索，提高响应速度

**技术方案**:
1. 创建 `cache/` 目录
2. 按日期缓存搜索结果
3. 缓存有效期：2 小时
4. 自动清理过期缓存

**文件结构**:
```
cache/
├── nba_2026-03-13.json
├── football_2026-03-13.json
└── f1_2026-03-13.json
```

---

#### P2-4: 错误处理优化 (1h)

**目标**: 更友好的错误提示和降级方案

**改进点**:
1. 搜索失败时显示友好提示
2. 网络超时时自动重试
3. 数据解析失败时跳过该项
4. 记录错误日志到文件
